# 业务流程分析文档

## 一、订单全流程

### 1.1 订单流程时序图

```mermaid
sequenceDiagram
    autonumber
    participant User as 用户
    participant BFF as BFF层<br/>(Gin:8888)
    participant Order as Order服务<br/>(gRPC:50051)
    participant DB as MySQL
    participant MQ as RabbitMQ
    participant Alipay as 支付宝

    rect rgb(240, 248, 255)
        Note over User, Alipay: 阶段一：创建订单
        User->>BFF: POST /notify/pay<br/>(订单请求)
        BFF->>Order: gRPC OrderAdd()
        
        Order->>Order: 生成订单号<br/>(时间戳+UUID前8位)
        
        loop 遍历商品列表
            Order->>DB: 查询商品信息
            DB-->>Order: 返回商品详情
            Order->>Order: 计算小计&累计总价
            Order->>Order: 构建订单明细
        end
        
        Order->>DB: 创建订单记录(Order表)
        Order->>DB: 创建订单明细(OrderItem表)
        
        par 异步库存预扣
            Order->>MQ: 发送库存扣减消息<br/>(stock_queue)
        and 生成支付链接
            Order->>Alipay: 请求生成支付页面
            Alipay-->>Order: 返回支付URL
        end
        
        Order-->>BFF: 返回订单信息<br/>(订单号/总金额/支付链接)
        BFF-->>User: 返回支付页面
    end

    rect rgb(255, 250, 240)
        Note over User, DB: 阶段二：支付与回调
        User->>Alipay: 完成支付
        Alipay->>BFF: 异步回调 POST /notify/pay
        BFF->>BFF: 解析回调参数
        BFF->>BFF: 验证trade_status
        
        BFF->>DB: 开启事务
        BFF->>DB: 查询订单
        BFF->>BFF: 幂等检查(PayStatus==2?)
        
        BFF->>DB: 更新订单状态(PayStatus=2)
        BFF->>DB: 查询订单明细
        
        loop 遍历订单明细
            BFF->>DB: 查询商品库存
            BFF->>DB: 扣减库存(Num -= quantity)
        end
        
        BFF->>DB: 提交事务
        BFF-->>Alipay: 返回success
    end
```

## 二、库存全流程

### 2.1 库存流程时序图

```mermaid
sequenceDiagram
    autonumber
    participant Order as Order服务
    participant MQ as RabbitMQ<br/>stock_queue
    participant Redis as Redis
    participant Consumer as 消费者服务
    participant DB as MySQL

    rect rgb(230, 255, 230)
        Note over Order, DB: 异步库存扣减(创建订单时)
        Order->>Order: 构建库存扣减消息<br/>{"goodsId":"1001","num":1}
        Order->>MQ: 发送消息到 stock_queue
        
        Note over MQ: 消息入队成功
        
        Consumer->>MQ: 监听 stock_queue
        MQ-->>Consumer: 推送消息
        
        Consumer->>Redis: 检查消息幂等性<br/>SETNX mq:msg:{hash}
        
        alt 消息未处理过
            Redis-->>Consumer: SETNX 返回 true
            Consumer->>Consumer: 执行库存扣减逻辑
            Note over Consumer: 当前仅记录日志<br/>未实际扣减库存
        else 消息已处理过
            Redis-->>Consumer: SETNX 返回 false
            Consumer->>Consumer: 过滤重复消息
        end
    end

    rect rgb(255, 240, 230)
        Note over Order, DB: 同步库存扣减(支付回调时)
        participant Callback as 支付回调
        
        Callback->>DB: 开启事务
        Callback->>DB: 查询订单明细
        DB-->>Callback: 返回明细列表
        
        loop 每个订单明细
            Callback->>DB: 查询商品库存
            DB-->>Callback: 返回当前库存
            Callback->>DB: UPDATE goods SET num = num - quantity
        end
        
        Callback->>DB: 提交事务
    end
```

### 2.3 ⚠️ 库存流程问题分析

**问题一：异步与同步库存扣减冲突**

| 阶段 | 操作 | 实际效果 |
|------|------|----------|
| 创建订单 | 发送MQ消息 | 消费者仅打印日志，未扣减 |
| 支付回调 | 事务扣减库存 | ✅ 实际扣减库存 |

**问题二：消费者未实现实际扣减**

```go
// 当前代码：仅打印日志
func ConsumeStockDeduct() {
    SubscribeMsg("stock_queue", func(msg string) {
        log.Println("执行库存扣减，消息：", msg)
        log.Println("库存扣减日志已记录")  // 未实际操作数据库
    })
}
```

**问题三：未支付订单的库存占用**

- 创建订单时不扣减库存 → 可能超卖
- 建议：创建订单时预扣库存，支付成功确认，超时未支付则释放

### 3.2 建议的积分流程设计

```mermaid
sequenceDiagram
    autonumber
    participant User as 用户
    participant Order as Order服务
    participant Points as 积分服务
    participant DB as MySQL
    participant MQ as RabbitMQ

    rect rgb(240, 240, 255)
        Note over User, MQ: 积分获取流程(支付成功后)
        Order->>MQ: 发送支付成功事件
        MQ->>Points: 消费支付成功消息
        
        Points->>Points: 计算积分<br/>积分 = 订单金额 × 积分比例
        
        Points->>DB: 查询用户积分账户
        Points->>DB: 更新积分余额(增加)
        Points->>DB: 记录积分流水(收入)
        
        Points-->>User: 积分到账通知
    end

    rect rgb(255, 255, 240)
        Note over User, DB: 积分使用流程(下单抵扣)
        User->>Order: 下单时选择积分抵扣
        Order->>Points: 查询用户可用积分
        Points-->>Order: 返回可用积分
        
        Order->>Points: 冻结积分
        Points->>DB: 积分余额减少
        Points->>DB: 记录积分流水(冻结)
        
        alt 支付成功
            Order->>Points: 确认扣减积分
            Points->>DB: 记录积分流水(支出)
        else 订单取消/超时
            Order->>Points: 释放冻结积分
            Points->>DB: 积分余额恢复
        end
    end
```

### 3.3 建议的积分数据模型

```sql
-- 积分账户表
CREATE TABLE points_account (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id INT NOT NULL COMMENT '用户ID',
    total_points INT DEFAULT 0 COMMENT '总积分',
    available_points INT DEFAULT 0 COMMENT '可用积分',
    frozen_points INT DEFAULT 0 COMMENT '冻结积分',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    UNIQUE KEY uk_user_id (user_id)
) COMMENT='积分账户表';

-- 积分流水表
CREATE TABLE points_log (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id INT NOT NULL COMMENT '用户ID',
    points INT NOT NULL COMMENT '积分变动数量(正数收入,负数支出)',
    type TINYINT NOT NULL COMMENT '类型:1收入 2支出 3冻结 4解冻',
    source VARCHAR(50) COMMENT '来源:订单支付/积分抵扣/系统赠送',
    order_no VARCHAR(32) COMMENT '关联订单号',
    balance_after INT COMMENT '变动后余额',
    remark VARCHAR(200) COMMENT '备注',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    KEY idx_user_id (user_id),
    KEY idx_order_no (order_no)
) COMMENT='积分流水表';
```
